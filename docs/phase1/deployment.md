# Deployment Guide

## Prerequisites
- Linux with KVM support (`/dev/kvm` must exist)
- Docker Engine 24+
- Docker Compose v2
- Rust toolchain (for building backend)
- Git

## Quick Start

### Setup (first time)

```bash
# Clone
git clone https://github.com/pruthvirajdgit/oneclick-ai.git
cd oneclick-ai

# Full machine setup (installs dependencies, builds rootfs, generates .env)
sudo ./scripts/setup/clean_setup.sh
```

`clean_setup.sh` handles:
- Installing Firecracker binary
- Setting up KVM permissions (udev rule)
- Creating TAP devices
- Building the rootfs template
- Generating `.env` with random secrets

### Start

```bash
./scripts/server/start.sh
```

This starts:
1. Docker services (frontend, PostgreSQL, Redis) via `docker compose up -d`
2. Backend (Rust binary) on the host, loading `.env` via dotenvy

### Stop

```bash
./scripts/server/stop.sh
```

### Access
- Frontend: http://localhost (port 80) or http://localhost:3000
- Swagger UI: http://localhost:8080/swagger-ui/
- Backend API: http://localhost:8080/api/
- PostgreSQL: localhost:5432 (user: oneclick, db: oneclick)
- Redis: localhost:6379

### Development workflow
```bash
# Rebuild frontend after code changes
docker compose up -d --build frontend

# Rebuild backend after code changes
cd backend && cargo build --release

# Build agent Docker image (for rootfs template)
docker build -t oneclick-agent:latest oneclick-runtime/

# Rebuild rootfs template after agent image changes
sudo bash scripts/firecracker/build-rootfs.sh

# Run database migrations
cd backend && sqlx migrate run

# Connect to database
docker compose exec postgres psql -U oneclick

# View backend logs
tail -f logs/*.log

# View Docker service logs
docker compose logs -f frontend
```

## Production (Azure VM)

### Server Requirements
- **Minimum**: 4 vCPU, 16GB RAM, 128GB SSD (Azure B4ms)
- OS: Ubuntu 24.04 LTS
- KVM support required (nested virtualization enabled if running in a VM)
- Docker Engine installed

### Deploy

```bash
# SSH to server
ssh user@<server-ip>

# Clone and setup
git clone https://github.com/pruthvirajdgit/oneclick-ai.git
cd oneclick-ai
sudo ./scripts/setup/clean_setup.sh

# Edit .env with production values (API keys, etc.)
nano .env

# Start with production config
docker compose -f docker-compose.yml -f docker-compose.prod.yml up -d
# Then start backend separately on the host
./scripts/server/start.sh

# Verify
curl http://localhost:8080/health
```

### Production .env

```bash
# Database
DATABASE_URL=postgres://oneclick:<strong-password>@localhost:5432/oneclick

# Auth (generated by clean_setup.sh)
JWT_SECRET=<random-64-char-string>

# LLM
GROQ_API_KEY=gsk_...
OPENROUTER_API_KEY=sk-or-v1-...

# Firecracker
AGENT_RUNTIME=firecracker
FC_KERNEL_PATH=/opt/firecracker/vmlinux-6.1
FC_ROOTFS_TEMPLATE=/opt/firecracker/rootfs-openclaw.ext4
FC_VCPU_COUNT=2
FC_MEM_SIZE_MIB=1536
FC_TAP_COUNT=4

# Limits
MAX_AGENTS=100
FREE_TIER_DAILY_LIMIT=50
IDLE_TIMEOUT_MINUTES=15
```

### docker-compose.prod.yml

Production overlay hides database ports (PostgreSQL, Redis not exposed externally):

```yaml
services:
  frontend:
    restart: always

  postgres:
    restart: always
    ports: !reset []    # Hide DB port in production

  redis:
    restart: always
    ports: !reset []    # Hide Redis port in production
```

> Note: For production TLS, add a reverse proxy (Caddy, nginx, or cloud load balancer) in front of the frontend container on port 80.

### Backups

```bash
# Database backup (daily cron)
0 3 * * * docker compose exec -T postgres pg_dump -U oneclick oneclick | gzip > /backups/db-$(date +\%Y\%m\%d).sql.gz
```

### Monitoring

```bash
# Health check
curl http://localhost:8080/health

# Resource usage
docker stats --no-stream

# Backend logs (daily rotating files)
ls -la logs/
tail -f logs/*.log

# Docker service logs
docker compose logs -f --tail 100 frontend
```

## Architecture Notes

- **Backend runs on the HOST**, not in Docker — it needs direct KVM access for Firecracker
- **Frontend, PostgreSQL, Redis** run in Docker via docker-compose.yml
- **No backend Dockerfile** — the backend is never containerized
- **No Traefik** — nginx inside the frontend container proxies `/api` to `host.docker.internal:8080`
- Backend auto-loads `.env` via dotenvy on startup
